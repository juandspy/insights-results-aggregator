This migration drops the userid columns from the advisorratings table. Some tables have the userid in the PRIMARY KEY, but we should be OK dropping it anyway because userid was actually account_number, so the tables shouldn't have duplicate records per rule (or per cluster) per organization. Organization IDs were added and populated in previous migrations.
