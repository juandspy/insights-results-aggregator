This migration deletes invalid rows from tables that were modified in mig0026 (added and populated orgid columns based on report table) -- clusterruletoggle, clusterruleuserfeedback and clusteruserruledisablefeedback. If the newly populated org_id columns are at the default value (0), that means we don't have that cluster in the report table (could happen, we sometimes don't check if a cluster exists), therefore they're invalid rows.
